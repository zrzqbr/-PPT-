## 一、背景规范（Background）【代码锁死 ⛔】

> **代码已锁死**：`scripts/apply_template.py` 中 `set_slide_background(slide, image_path)` 函数硬编码了背景铺满逻辑（全屏图片 + z-order 最底层），所有适配流程必须通过此函数设置背景。

### 1.1 背景资产

| 文件 | 用途 | 分辨率 | 设计描述 |
| --- | --- | --- | --- |
| `assets/backgrounds/bg-cover.jpeg` | 封面 / 章节扉页 / 结尾页 | 1920×1080 | 米黄渐变底，红色细线稿城市天际线（含古建筑、起重机、烟花），左上角红色半调网点装饰（长沙城市主题） |
| `assets/backgrounds/bg-content.jpeg` | 内容页（正文/列表/表格） | 1920×1080 | 淡米黄底，浅红色低饱和水印城市线稿，整体不抢视觉，适合承载文字与图表 |

### 1.2 应用规则

1. **铺满整页**：背景图必须等比例铺满整个幻灯片（Cover 模式），不允许平铺、不允许镂空。
2. **不可加滤镜**：不允许加模糊、暗化、变色、半透明遮罩。
3. **不可换色**：禁止把红色城市线稿换成其他颜色。
4. **例外情况**：
   - 全屏图表 / 全屏代码 / 全屏截图：允许使用纯白 `#FFFFFF` 背景，便于阅读；此时角标 Logo 必须保留。
   - 视频嵌入页：允许纯黑 `#000000` 背景。
   - 不允许其他纯色背景或第三方背景图。

### 1.3 页面类型识别（自动判断）

适配器在迁移已有 PPT 时，按以下优先级判断页面类型（v7 P1-A 增强）：

**优先级 1：Slide Layout 名称匹配**（最可靠）

| Layout 名称关键词 | 页面类型 |
| --- | --- |
| 封面、cover、title slide、首页 | cover |
| 章节、section、扉页、divider、break | section |
| 结尾、ending、结束、thank、end、封底 | end |

**优先级 2：首尾页默认规则**

| 判断条件 | 页面类型 | 背景 |
| --- | --- | --- |
| 第 1 页 | cover（封面） | bg-cover |
| 最后 1 页 | end（结尾页） | bg-cover |

**优先级 3：内容启发式**

| 判断条件 | 页面类型 | 背景 |
| --- | --- | --- |
| 单页只有 1–2 个超大标题文本（>=36pt）、无其他形状 | section（章节扉页） | bg-cover |
| 含图表 / 表格 / 多文本框 / 多形状 | content（内容页） | bg-content |
| 含全屏图片或视频 | content（特殊） | 保留原图，或使用 bg-content |

如果无法判定，默认使用 `bg-content`。

## 二、字体规范（Typography）

### 2.1 字体族（强制锁定，无备选）

> **v2 修订**：字体回退链已废弃。中英文统一使用腾讯体，不再维护多字体回退。

| 用途 | **必须使用的 family name** | 说明 |
| --- | --- | --- |
| 中文 / 西文标题、强调 | **`TencentSans W7`** | 腾讯体粗体，覆盖中英文字符 |
| 中文 / 西文正文 | **`TencentSans W3`** | 腾讯体常规，覆盖中英文字符 |
| 代码 / 等宽 | `JetBrains Mono` / `Consolas` / `Courier New` | 仅代码块，保留不替换 |

**⚠️ 禁止使用的字体名**（会导致字体识别失败或显示为方块）：
- `腾讯体 W7` / `腾讯体 W3`（中文别名，仅中文版 Office 识别，跨平台不安全）
- `TencentSansW7` / `TencentSansW3`（无空格的 PostScript 名，PowerPoint 不识别）
- `TTTGB Medium`、`Helvetica`、`Arial`、`PingFang SC`、`PingFang SC Semibold`、`微软雅黑`、`思源黑体`（一律禁止）

### 2.2 字号建议

| 元素 | 推荐字号 | 最小字号 |
| --- | --- | --- |
| 封面主标题 | 60–88pt | 48pt |
| 封面副标题 | 24–32pt | 20pt |
| 章节扉页标题 | 48–60pt | 40pt |
| 内容页大标题 | 32–40pt | 28pt |
| 内容页小标题 | 22–28pt | 20pt |
| 正文 | 18–22pt | 16pt |
| 注脚 / 来源 | 12–14pt | 12pt |
| 表格内文 | 14–18pt | 12pt |

**铁律**：任何文本 ≥ 12pt。低于 12pt 必须放大。

### 2.3 字色

- 标题：`#000000` 或 `#D80C01`（品牌红，用于强调）
- 正文：`#222222`（深灰，比纯黑柔和）
- 注脚：`#888888`
- 强调高亮：`#FF0000`（品牌红）

### 2.4 字体替换策略（v2 — 无回退链）

遍历每个 `a:rPr` 节点，根据其上下文判断用途后，**直接写入唯一指定字体**，不设回退链：

```
if 文本 run 在标题占位符 OR 字号 >= 24pt:
    latin / ea / cs → TencentSans W7
else:
    latin / ea / cs → TencentSans W3
```

不要替换：
- 已经使用代码字体（JetBrains Mono / Consolas / Menlo / Courier）的代码块。
- 数学公式字体（Cambria Math）。

> **注意**：`a:latin`、`a:ea`、`a:cs` 三个节点都必须统一写入，否则某些 Office 版本会用 fallback 字体覆盖 ea/cs。

## 四、配色规范（与腾讯云架构师长沙同盟模板背景安全协作）

> 腾讯云架构师长沙同盟模板背景是**米黄底 + 红色城市线稿**的非中性视觉，任何上游 PPT Skill 生成的色块、卡片、图表配色，都必须从下列**安全调色板**中取色，避免与背景撞车。

### 4.1 主色系（Brand Core）— 与模板天然搭配

| 颜色 | HEX | 用途 |
| --- | --- | --- |
| 品牌红（深） | `#D80C01` | 主标题、强调文字、关键数据、CTA 按钮 |
| 品牌红（亮） | `#FF0000` | 高亮、警示、Hover 状态 |
| 经典黑 | `#000000` | 文字主色 |
| 深灰文字 | `#222222` | 正文（比纯黑柔和） |
| 中灰 | `#666666` | 副文本、注释 |
| 浅灰 | `#888888` | 注脚、来源 |
| 纯白 | `#FFFFFF` | 卡片底、托盘 |
| 米白底 | `#FAF6EE` | 与模板背景同色系的卡片底 |
| 浅边线灰 | `#E7E6E6` | 分割线、表格边线 |

### 4.2 辅助色系（Accent，最多取 2 个，暖色优先）— 已校验与红色不撞

> ⚠️ `#08194B` 沉稳深蓝已移入禁用色（§4.5），内容色块必须优先选暖色系。

| 颜色 | HEX | 适用 |
| --- | --- | --- |
| 暖淡黄 | `#FAD16A` | 关键数据强调、强调色块（替代品牌红用于多系列对比，首选暖色） |
| 提示金 | `#F19D19` | 警告、过渡步骤 |
| 商务蓝 | `#1D6FA9` | 数据系列、流程图（非色块背景） |
| 高级灰绿 | `#4A5D52` | 中性图表配色 |

**⚠️ 禁止使用的颜色**（与红色背景撞色或破坏品牌调性）：
- 深蓝色（含 `#08194B`，已从品牌色移入禁用色）
- 鲜亮绿色（如 `#00FF00`、`#1D9A78`）— 红配绿大忌
- 青色 / Teal（如 `#00CED1`）
- 粉色 / 玫红（与品牌红冲突）
- 任何彩色渐变（蓝紫渐变、青绿渐变、彩虹渐变）
- 霓虹/赛博朋克高饱和荧光色

### 4.3 数据可视化标准色板（图表多系列）

按系列顺序优先使用：

```
系列 1：#D80C01   品牌红
系列 2：#D4A574   暖杏色（替代原 #08194B 深蓝）
系列 3：#FAD16A   暖淡黄
系列 4：#666666   中灰
系列 5：#1D6FA9   商务蓝
系列 6：#F19D19   提示金
```

**不要超过 6 个系列**。如果系列更多，建议拆图表，或使用同色系明度变化。

### 4.4 卡片 / 色块 / 容器配色规则

为避免与红色背景撞色，AI 生成的内容卡片、色块、容器必须遵守：

| 元素 | 建议背景 | 建议文字 |
| --- | --- | --- |
| 普通内容卡片 | `#FFFFFF` 或 `#FAF6EE` | `#222222` |
| 强调卡片 | `#D80C01` | `#FFFFFF` |
| 数据卡片 | `#D4A574` | `#FFFFFF` |
| 引用 / 提示框 | `#FAF6EE` + 左侧 `#D80C01` 实线 4px | `#222222` |
| 代码块 | `#000000` 或 `#1E1E1E` | `#FFFFFF` + 系列高亮 |

**禁止做的卡片样式**：
- 半透明 / 玻璃拟态（会透出红色城市线稿，文字看不清）
- 彩色渐变填充
- 多色描边
- 五颜六色的标签云

### 4.4-WarmTone 暖色调内容色块专区（扩展）

> 本节为内容型色块（信息卡、标签、步骤框、分类块）提供一套**与模板背景协调的暖色系**，与 4.4 基础规则并列使用。AI 生成色块时，应根据内容语义自主搭配，不必逐色指定。

#### 暖色调色板（5色）

| 色名 | HEX | 亮度感知 | 推荐文字色 |
| --- | --- | --- | --- |
| 珊瑚粉 | `#E8A090` | 中亮 | `#000000`（黑） |
| 暖杏色 | `#D4A574` | 中 | `#000000`（黑） |
| 浅金橙 | `#E8B84A` | 亮 | `#000000`（黑） |
| 玫瑰木 | `#C4A484` | 中暗 | `#000000`（黑）或 `#FFFFFF`（白） |
| 浅赭色 | `#D49A6A` | 中 | `#000000`（黑） |

**文字混搭规则（黑白选择依据）**：

```
if 色块背景亮度（L* in CIELAB）>= 55:
    文字色 → #000000（黑）
else:
    文字色 → #FFFFFF（白）
    # 玫瑰木 #C4A484 处于边界，L*≈62，优先黑色；
    # 如与深色相邻卡片对比需要时可用白色
```

**AI 配色决策逻辑（根据内容语义自动选色）**：

```
内容类型 → 建议色块
─────────────────────────────────────────────
步骤 / 流程（1-2-3顺序）   → 珊瑚粉 #E8A090（起始）
                              暖杏色 #D4A574（中间）
                              浅赭色 #D49A6A（结束）
优先级标签 / 警告提示       → 浅金橙 #E8B84A（醒目但不撞品牌红）
分类 / 归纳（并列卡片）     → 从5色中交替取色，相邻不重复
数据指标卡片               → 玫瑰木 #C4A484（沉稳衬托数字）
引用 / 小贴士              → 暖杏色 #D4A574 + 左侧 #D80C01 实线 4px
```

**⚠️ 暖色调专区使用约束**：
1. **单页最多使用 3 种暖色**，避免视觉混乱。
2. **不与品牌红 `#D80C01` 并列做相邻卡片**（两者都是暖红系，会互相竞争）；如需强调可用品牌红做描边线条而非填充。
3. **暖色色块内不放白色文字以外的彩色文字**；文字仅允许 `#000000` / `#FFFFFF` / `#D80C01`。
4. 暖色系色块不适合用作封面/章节扉页大面积背景，仅限内容页局部色块。

### 4.5 透明度规则

- 卡片不透明度 ≥ 95%（在 bg-content 上）
- Logo 托盘允许 60% 白色半透明
- 装饰性图形允许 20–40% 透明度
- **正文文字必须 100% 不透明**

### 4.6 上游 PPT Skill 的"配色指令"

当本 Skill 与其他 PPT 生成 Skill 串联时，在调用上游 Skill 前必须把下述指令注入 prompt：

```
配色约束（强制遵守）：
- 主色：#D80C01（品牌红）、#000000、#FFFFFF
- 辅助色（≤2种，优先暖色系）：#FAD16A（暖淡黄，首选）、#F19D19（提示金）、#1D6FA9（商务蓝，仅数据系列/流程图）
- #08194B 深蓝已弃用为禁用色，禁止用作色块/卡片背景
- 暖色调内容色块（优先使用，最多 3 种）：
    珊瑚粉 #E8A090、暖杏色 #D4A574、浅金橙 #E8B84A、
    玫瑰木 #C4A484、浅赭色 #D49A6A
  → 根据内容语义自主搭配（见 4.4-WarmTone 决策逻辑）
  → 暖色色块内文字仅用 #000000 / #FFFFFF / #D80C01
- 数据可视化按 4.3 节标准色板（#D80C01 → #D4A574 → #FAD16A → #666666 → #1D6FA9 → #F19D19）
- 数据卡片：暖杏色 #D4A574 底 + #FFFFFF 字（替代原 #08194B）
- 禁用：深蓝（含 #08194B）、蓝紫、绿色、青色、粉色、彩色渐变、玻璃拟态、霓虹
- 卡片不透明度 ≥ 95%
- 任何色块面积超过整页 30% 时，必须使用上述安全色之一
```

## 六、迁移增强能力（v7 新增）

### 6.1 文字色对比度修复（P0-A）

**问题**：原 PPT 为深色背景+白色文字，换成模板浅色米黄背景后白字完全不可读。

**解决**：`fix_text_contrast(slide)` 在背景替换后自动执行：
- 计算背景相对亮度（WCAG 2.1）
- 遍历所有文字 run（含表格、组合形状递归）
- 浅色背景（亮度 > 0.4） + 浅色文字（亮度 > 0.55） + 对比度 < 3:1 → 改为 `#222222`
- 深色背景（亮度 ≤ 0.4） + 深色文字（亮度 < 0.2） + 对比度 < 3:1 → 改为 `#FFFFFF`

适配报告输出 `contrast_fixes` 统计修复数量。

### 6.2 全屏遮罩清除增强（P0-B）

v7 扩展 `remove_fullscreen_overlays()` 支持检测：
- 实心色块遮罩（SOLID）
- **渐变填充遮罩（GRADIENT）**—— v7 新增
- **图案填充遮罩（PATTERNED）**—— v7 新增
- 背景填充遮罩（BACKGROUND）
- 图片遮罩（仅 z-order 底层前 2 个）

同时 `set_slide_background()` 在插入背景图前直接移除整个 `<p:bg>` 元素，覆盖所有填充类型。

### 6.3 智能页面分类（P1-A）

`classify_page()` 三级优先级：
1. **Slide Layout 名称**（最可靠）：匹配封面/章节/结尾关键词
2. **首尾页规则**：第 1 页 → cover，最后 1 页 → end
3. **内容启发式**：文本数量+大标题字号+图表判断

### 6.4 元素溢出检测（P1-B）

`check_element_overflow(slide)` 在 Logo 插入后执行：
- 检测内容是否溢出页面右/下边界（允许 0.1" 容差）
- 检测有文字的形状是否侵入 Logo 安全区（矩形相交检测）
- **仅报告不修改**，输出到适配报告 `overflow_warnings` 字段
